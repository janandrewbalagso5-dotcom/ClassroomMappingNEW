✦ What This System Does
🗺️
Every room has a story. We make sure no two stories clash.
The system tracks every room across 3 floors (Ground, 2nd, 3rd) and instantly detects if a room, instructor, or section is double-booked — before it's saved.
📅
MWF or TTH — we've got both covered, all in one grid.
Schedules are split into MWF (Monday-Wednesday-Friday) and TTH (Tuesday-Thursday) day types, rendered as an interactive room-by-timeslot table you can click to edit inline.
📊
Teaching is more than showing up — this system proves it on paper.
The Teaching Load Report auto-generates a printable document per instructor: subjects, schedules, units, room numbers, and a signature block — ready for official use.
✦ Did You Know?
Security
Passwords are never stored as plain text.
Every password is scrambled through SHA-256 hashing — a 64-character code — before it ever touches the database. Even admins can't see your real password.
Registration
Students & Instructors sign up themselves. Admins don't.
Anyone can register as a Student or Instructor. Admin and Dean accounts are sealed — they must be created directly inside the system by an existing Admin.
Conflict Guard
3 conflicts are checked before any schedule is saved.
Room conflict, Instructor conflict, and Section conflict — all three run every time a slot is assigned or updated, keeping the schedule clean and honest.
Student View
Students only see the instructors from their own course.
When a student logs in, the instructor dropdown on their map is automatically filtered to match their registered course code — no manual setup needed.
Smart Routing
The system sends you to the right page the moment you log in.
Students land on StudentHome.aspx. Staff (Admin, Dean, Instructor) go to the full dashboard. Every role has its own front door.
Instructor Tip
Instructors can only assign themselves — no impersonating others.
When adding a schedule entry, Instructors can only pick themselves as the instructor. They also see only their own subjects in every dropdown.
✦ One Line Per Role
👑
Admin
"The only one with all the keys."
Full access to everything — schedules, subjects, instructors, and user accounts. The only role that can create or delete other users.
🎓
Dean
"Almost all access — except managing users."
Can handle everything academic — subjects, instructors, and schedules — but user account management stays in Admin hands only.
👨‍🏫
Instructor
"My schedule, my subjects, my lane."
Instructors manage only their own entries. They can't edit another instructor's schedule, assign other people to their subjects, or touch user accounts.
🎒
Student
"Look, but don't touch."
Read-only access to a room map filtered to their course's instructors. Any attempt to access management pages redirects to Access Denied.
✦ Good to Know
⏱️
Idle for 30 minutes? The system will log you out automatically. Session is stored in server memory — not in your browser or a cookie file.
🏫
Rooms 206 and 207 are computer labs. The system labels them "206 (LAB)" and "207 (LAB)" so they're easy to spot on the classroom map.
🔗
Schedules are linked to instructors by name — not by ID. If an instructor's name is changed, their old schedule entries won't update automatically. Best to keep names consistent.
📋
When an Instructor registers, the system automatically derives the correct course code from the department they choose — server-side, so it can't be tampered with through the browser.
🖨️
Teaching Load Reports match subjects to schedules using the Subject Code. If a code is slightly different between the two tables, the Days/Time/Room columns will show up blank in the printed report.
🔐
The system uses Windows Integrated Security to connect to the database — no username or password is stored in the config file. Clean and secure by design.
